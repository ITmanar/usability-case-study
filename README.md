# Mobile App Usability Case Study – McDelivery KSA

## Overview
This repository contains an academic usability evaluation of the McDelivery KSA mobile application, conducted as part of the IT215 – Human Computer Interaction course.

The study focuses on identifying usability issues based on established HCI principles and observing real user interaction with the application.

---

## Scope of the Study
- Interface analysis using HCI and usability principles
- Heuristic evaluation
- Task-based usability testing with participants
- Identification of usability problems
- Practical design recommendations

---

## Key Usability Issues Identified
- Inconsistent visual design for primary action buttons
- Slow system response with lack of feedback
- High cognitive load due to poor information organization
- Mixed-language interface elements
- Small and closely placed buttons affecting accessibility

---

## Methods Used
- Heuristic Evaluation (Nielsen’s heuristics)
- Cognitive Walkthrough
- Usability Testing with real users
- Metrics: task completion time, error rate, user satisfaction

---

## Recommendations
- Standardize CTA button colors
- Add loading indicators and system feedback
- Improve visual hierarchy and reduce clutter
- Maintain consistent interface language
- Improve button size and spacing for better accessibility
